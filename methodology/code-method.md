# CODE Method

> The four-stage workflow underneath [Second Brain](./second-brain.md): **C**apture · **O**rganise · **D**istil · **E**xpress.

## Summary

CODE is Tiago Forte's name for the lifecycle every piece of information should travel through if it's going to earn its keep in a knowledge base. Each stage has a distinct job, a distinct trigger, and a distinct quality bar. Most failed PKM systems collapse the stages — they capture and organise simultaneously, or expect distillation to happen by inspiration alone.

## The four stages

### 1. Capture — *what resonates*

- **Trigger:** something you read, heard, or thought makes you nod, want to argue with it, or solves a problem you have right now.
- **Action:** quick-capture into the vault using the [knowledge capture template](../templates/knowledge-capture-template.md). Inbox-style, no organising.
- **Quality bar:** *resonance*, not completeness. If you don't react to it, skip it.
- **Tools in this vault:** raw scratch lives in [daily notes](../templates/daily-note-template.md); web/article clippings in `resources/` with a 2-line "why it matters."

### 2. Organise — *by actionability*

- **Trigger:** you sit down for the [weekly review](./daily-weekly-review.md), or a fresh capture is obviously project-bound.
- **Action:** file the capture into one of the four [PARA](./para-method.md) buckets — the question is "*what's this for?*", not "*what's this about?*".
- **Quality bar:** the location predicts when you'll see it again. A capture filed under a current Project should resurface this week. One filed in Resources may not resurface until a future project pulls it in.

### 3. Distil — *progressively*

- **Trigger:** you re-encounter a captured note, or you're preparing to use it in output.
- **Action:** apply [progressive summarization](./progressive-summarization.md) — bold the load-bearing lines, highlight the bolded essentials, write a 2-sentence executive summary on top.
- **Quality bar:** *future-you opens the note and gets the gist in 5 seconds.*
- **Vault practice:** a distilled concept usually graduates from `resources/` into [concepts/](../concepts/index.md) as an [evergreen note](../concepts/evergreen-notes.md).

### 4. Express — *ship something*

- **Trigger:** a project needs an output: a PRD, a doc, a talk, a snippet, a decision.
- **Action:** pull from the distilled layer to compose the artifact. Cite the captures that informed it.
- **Quality bar:** something *leaves the vault*. If your knowledge base never produces output, it isn't a Second Brain — it's a hoard.

## A concrete pass

You see a tweet about [progressive summarization](./progressive-summarization.md):

1. **Capture** — paste it into today's [daily note](../templates/daily-note-template.md) with one line of why it caught your eye.
2. **Organise** — Friday review: it isn't tied to a current project; file it under Resources as `resources/progressive-summarization-tweet.md`.
3. **Distil** — two weeks later, while writing this very page, you re-read it, bold the load-bearing claims, and lift the core idea into the [progressive-summarization.md](./progressive-summarization.md) concept note.
4. **Express** — the concept ships as part of this methodology folder, cited back to the original tweet.

## Where teams trip up

- **Skipping Distil.** People go straight from Capture to Express, hunting the inbox for the right snippet under deadline. The distilled layer is what makes the system fast.
- **Capturing for completeness.** The vault is not a personal Wikipedia; aim for *high-resonance density*, not coverage.
- **Re-organising as procrastination.** Constant folder-shuffling masquerades as productive work. Use the [weekly review](./daily-weekly-review.md) cadence instead of *ad hoc* tidying.
