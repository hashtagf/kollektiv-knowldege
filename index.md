# Kollektiv Knowledge Base

Welcome to the Kollektiv Obsidian-style knowledge base. This is the home node — use it to navigate the vault.

## Navigate

- [Concepts](./concepts/index.md) — ideas, theories, definitions, frameworks
- [Projects](./projects/index.md) — active and archived project notes
- [People](./people/index.md) — colleagues, contacts, stakeholders
- [Resources](./resources/index.md) — articles, books, links, references
- [Templates](./templates/index.md) — starter notes for new entries

## How to use this vault

1. Open this folder in [Obsidian](https://obsidian.md) (or any markdown editor).
2. Start a new note by copying a file from [Templates](./templates/index.md).
3. Place the note in the matching folder (concept, project, person, or resource).
4. Link related notes with `[[wiki-links]]` or `[relative markdown links](./concepts/index.md)`.
5. Add tags in the frontmatter so notes cluster by topic.

## Conventions

- **Filenames:** lowercase, hyphen-separated (`my-new-concept.md`).
- **Frontmatter:** every note begins with YAML frontmatter (`title`, `tags`, `created`).
- **Linking:** link generously. A good note links to at least two other notes.
- **Index files:** every folder has an `index.md` that lists its contents.

See [README](./README.md) for full usage instructions.
